# OPC Gateway Tool 使用說明書

## 簡介

OPC Gateway Tool 是一個將 OPC DA (OLE for Process Control Data Access) 轉換為 OPC UA (OPC Unified Architecture) 的閘道器工具。它能夠連接到現有的 OPC DA 伺服器，讀取數據並將其透過內建的 OPC UA 伺服器提供給現代的 OPC UA 客戶端。

## 系統需求

### 作業系統
- Windows 7 或更新版本
- 支援 .NET Framework 4.8

### 硬體需求
- CPU: 1 GHz 或更快的處理器
- 記憶體: 最少 512 MB RAM
- 硬碟空間: 100 MB 可用空間

### 必要的執行環境
- **.NET Framework 4.8**: 如果系統未安裝，請從 Microsoft 官網下載安裝
- **OPC Core Components**: 用於 OPC DA 通訊
  - 建議安裝 OPC Core Components 3.0 或更新版本
  - 可從 OPC Foundation 官網下載

## 安裝與部署

### 1. 檔案部署
將編譯後的所有檔案複製到目標目錄，確保包含以下重要檔案：
```
OPCGatewayTool.exe          # 主程式
NLog.config                 # 日誌配置檔
Config/                     # 配置檔目錄
Logs/                      # 日誌檔目錄
Resources/                 # 資源檔目錄
[其他 DLL 檔案]            # 相依性函式庫
```

### 2. 權限設定
- 確保應用程式具有讀寫權限
- 建議以系統管理員權限執行（特別是首次執行時）

## 操作流程

### 第一步：啟動應用程式
1. 執行 `OPCGatewayTool.exe`
2. 應用程式將自動載入預設配置

### 第二步：配置 OPC DA 連線
1. 在「OPC DA 配置」區塊中：
   - **伺服器名稱**: 輸入目標 OPC DA 伺服器的 ProgID（例如：`Matrikon.OPC.Simulation.1`）
   - **更新頻率**: 設定數據更新間隔（毫秒，預設 1000ms）
   - **主機名稱**: 如果 OPC DA 伺服器不在本機，請輸入遠端主機名稱

2. 點擊「連接 OPC DA」按鈕

3. 連接成功後，左側樹狀檢視將顯示可用的 OPC 項目

### 第三步：瀏覽和選擇數據項目
1. 展開左側的 OPC 項目樹狀結構
2. 瀏覽並選擇需要映射的數據項目
3. 選中的項目將顯示當前值和品質狀態

### 第四步：建立數據映射
1. 選擇要映射的 OPC DA 項目
2. 點擊「新增映射」或使用右鍵選單
3. 系統將自動為選中的項目建立 OPC UA 節點映射

### 第五步：配置和啟動 OPC UA 伺服器
1. 在「OPC UA 配置」區塊中：
   - **伺服器名稱**: 設定 OPC UA 伺服器名稱
   - **端口**: 設定 OPC UA 伺服器監聽端口（預設 4840）
   - **最大客戶端數**: 設定允許的最大連線數

2. 點擊「啟動 OPC UA」按鈕

3. 啟動成功後，端點 URL 將顯示為 `opc.tcp://localhost:端口號`

### 第六步：驗證連線
1. 使用 OPC UA 客戶端工具（如 UAExpert）連接到閘道器
2. 瀏覽 Objects → Gateway 節點下的映射項目
3. 確認數據能正常讀取和更新

## 配置檔說明

### 主要配置檔
應用程式會在 `Config/` 目錄下自動產生配置檔，包含：
- OPC DA 連線設定
- OPC UA 伺服器設定
- 數據映射關係

### NLog.config（日誌配置）
```xml
<?xml version="1.0" encoding="utf-8" ?>
<nlog>
  <targets>
    <target xsi:type="File" name="fileTarget"
            fileName="Logs/${shortdate}.log"
            createDirs="true" />
  </targets>
  <rules>
    <logger name="*" minlevel="Info" writeTo="fileTarget" />
  </rules>
</nlog>
```

## 常見問題與排除

### Q1: OPC DA 連線失敗
**可能原因與解決方案：**
- 確認 OPC DA 伺服器正在運行
- 檢查 OPC Core Components 是否正確安裝
- 確認 ProgID 名稱正確
- 以系統管理員權限執行

### Q2: OPC UA 伺服器無法啟動
**可能原因與解決方案：**
- 檢查端口是否被其他程式佔用
- 確認防火牆設定允許該端口
- 檢查日誌檔案中的錯誤訊息

### Q3: 數據不更新
**可能原因與解決方案：**
- 檢查 OPC DA 項目是否有效
- 確認數據映射關係正確建立
- 檢查 OPC DA 伺服器的數據品質

### Q4: 客戶端無法連接到 OPC UA 伺服器
**可能原因與解決方案：**
- 確認端點 URL 正確
- 檢查網路連線
- 確認 OPC UA 伺服器已成功啟動

## 日誌系統

### 日誌位置
- 應用程式執行目錄下的 `Logs/` 資料夾
- 日誌檔案命名格式：`YYYY-MM-DD.log`

### 日誌級別
- **Trace**: 詳細的追蹤訊息
- **Debug**: 除錯訊息
- **Info**: 一般資訊訊息
- **Warn**: 警告訊息
- **Error**: 錯誤訊息

### 日誌檔案管理
- 每日自動輪替日誌檔案
- 保留最近 7 天的日誌檔案
- 舊檔案自動壓縮儲存

## 效能調校建議

### OPC DA 設定
- **更新頻率**: 根據實際需求調整，過於頻繁會影響效能
- **項目數量**: 建議單次映射項目數不超過 1000 個

### OPC UA 設定
- **最大客戶端數**: 根據伺服器效能調整
- **會話超時**: 適當設定以平衡效能和穩定性

### 系統效能
- 定期清理日誌檔案
- 監控記憶體使用情況
- 確保網路頻寬充足

## 安全性考量

### 網路安全
- 在生產環境中請配置適當的防火牆規則
- 考慮使用 VPN 或其他安全通道
- 定期更新系統和相依性元件

### 存取控制
- OPC UA 支援匿名連線（預設設定）
- 可根據需要配置使用者驗證和權限控制

## 程式關閉保護機制

### 自動清理功能
應用程式具備完善的防呆機制，確保關閉時不會造成連線卡住：

### 關閉確認對話框
- 當有 OPC 服務正在運行時，關閉程式會彈出確認對話框
- 提醒使用者將自動停止所有連線後關閉
- 可選擇取消關閉操作

### 自動清理順序
程式關閉時會按以下順序自動清理：
1. **停止 OPC UA 伺服器** - 斷開所有客戶端連線
2. **斷開 OPC DA 連線** - 釋放 OPC DA 伺服器資源
3. **清理內部資源** - 釋放記憶體和系統資源
4. **等待清理完成** - 確保所有資源正確釋放

### 日誌記錄
所有清理過程都會記錄在日誌檔案中，便於追蹤和除錯：
```
MainViewModel 正在清理資源...
自動停止OPC UA伺服器
自動斷開OPC DA連線
MainViewModel 資源清理完成
```

### 異常處理
即使清理過程中發生錯誤，程式也會：
- 記錄詳細的錯誤資訊
- 繼續執行其他清理步驟
- 確保程式能正常退出

**注意：建議正常情況下手動停止服務後再關閉程式，以獲得最佳體驗。**
